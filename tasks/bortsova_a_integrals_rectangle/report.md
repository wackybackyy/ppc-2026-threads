## 1. Введение

Численное интегрирование является одной из задач вычислительной математики.
Метод прямоугольников позволяет вычислить определенный интеграл функции одной
или нескольких переменных. При увеличении размерности и числа шагов разбиения вычислительная
сложность растет экспоненциально, что делает задачу хорошей для распараллеливания.

Цель работы: реализовать последовательный алгоритм вычисления многомерного интеграла методом
прямоугольников и его параллельные версии с использованием технологий OpenMP,
Intel Threading Building Blocks, std::thread и MPI + OpenMP.

## 2. Постановка задачи

**Вход:**

- Подынтегральная функция `f(x1, x2, ..., xn)`
- Нижние и верхние границы интегрирования для каждой размерности
- Число шагов разбиения `N` (одинаковое для всех размерностей)

**Выход:**

- Приближенное значение многомерного интеграла

**Ограничения:**

- Размерность >= 1
- Число шагов > 0
- Функция должна быть задана

## 3. Базовый алгоритм

1. Для каждой размерности вычисляется шаг разбиения `h_i = (b_i - a_i) / N`.
2. Вычисляется объем элементарного прямоугольника: `V = h_1 * h_2 * ... * h_n`.
3. Для каждой размерности предвычисляются координаты средних точек: `x_{i,j} = a_i + (j + 0.5) * h_i`.
4. Перебираются все `N^n` точек сетки. Для каждой точки вычисляется значение функции.
5. Результат: `I = V * sum(f(x))` по всем точкам.

Общее число точек `N^n`, где `n` — размерность. Сложность: `O(N^n)`.

## 4. Схема распараллеливания

### 4.1 OMP

Используется директива `#pragma omp parallel for reduction(+:sum)` для параллелизации
основного цикла по точкам. Каждый поток обрабатывает свой диапазон индексов и вычисляет
частичную сумму, которая затем складывается через reduction.

### 4.2 TBB

Используется `tbb::parallel_reduce` с `tbb::blocked_range<int64_t>`. TBB автоматически
разбивает диапазон точек на блоки и распределяет их между потоками. Каждый блок вычисляет
свою частичную сумму, результаты объединяются через `std::plus<>`.

### 4.3 STL

Пространство точек делится на равные чанки между потоками. Главный поток выполняет свою
часть работы, а остальные `(num_threads - 1)` потоков создаются через `std::thread`.
Каждый поток записывает свою частичную сумму в отдельную ячейку вектора `partial_sums`.
После завершения всех потоков результаты суммируются.

### 4.4 ALL

Двухуровневая параллелизация:

- **MPI:** Общее пространство точек делится между MPI-процессами. Каждый процесс получает
  свой диапазон индексов. После вычисления `MPI_Reduce` собирает частичные суммы на
  процессе 0, затем `MPI_Bcast` рассылает результат всем процессам.
- **OpenMP:** Внутри каждого MPI-процесса работа дополнительно распараллеливается
  между потоками через `#pragma omp parallel`.

## 5. Детали реализации

### Структура файлов

- `common/include/common.hpp` — общие типы (`InType`, `OutType`, `Function`)
- `seq/` — последовательная реализация
- `omp/` — OMP реализация
- `tbb/` — TBB реализация
- `stl/` — STL реализация
- `all/` — MPI + OpenMP реализация
- `tests/functional/main.cpp` — функциональные тесты (10 тестовых случаев)
- `tests/performance/main.cpp` — тесты производительности

### Ключевые классы

Каждая технология реализует класс `BortsovaAIntegralsRectangle<TECH>`, наследующий `BaseTask`,
с методами:

- `ValidationImpl()` — проверка корректности входных данных
- `PreProcessingImpl()` — предвычисление средних точек и объема
- `RunImpl()` — основные вычисления с параллелизацией
- `PostProcessingImpl()` — завершающая обработка

В STL и ALL реализациях вычисление частичной суммы вынесено в отдельный метод
`ComputePartialSum()` для снижения когнитивной сложности.

### 7.3 Анализ

- **TBB** демонстрирует наилучшую масштабируемость среди потоковых технологий
  с эффективностью 87–95%.
- **STL** показывает хорошее ускорение на малом числе потоков (до 5), но overhead
  от создания потоков снижает эффективность при 7 потоках.
- **ALL** хорошо масштабируется при увеличении числа MPI-процессов с 1 до 7 при
  фиксированном числе потоков. При большом числе процессов и потоков одновременно
  эффективность падает из-за конкуренции за ресурсы CPU.
- Во всех технологиях режим **task_run** стабильно быстрее **task_pipeline**.

## 8. Выводы

Реализованы и протестированы 5 версий алгоритма вычисления многомерного интеграла
методом прямоугольников. Все версии дают корректные результаты. Параллельные реализации
обеспечивают значительное ускорение на многоядерных системах. TBB показал наилучшее
соотношение ускорения и простоты использования. Комбинация MPI + OpenMP позволяет
масштабировать вычисления как по процессам, так и по потокам.
